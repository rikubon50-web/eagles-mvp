# ロースターを「学年(1〜4)」から「期(37期…)」表示へ — 設計

## 背景・目的

現状ロースターは部員の `year`（学年 1〜4）でグルーピングし「1ST/2ND/3RD/4TH」と表示している。
学年は毎年4月に全員分を手で更新する必要があり保守コストが高い。

「期（37期・38期…）」は一度決まれば一生変わらない不変値。各部員に期だけを持たせ、
**現役判定（1〜4年相当か）と表示は今日の日付から自動計算**することで、毎年の更新を不要にする。

## 確定した前提（アンカー）

- **2026年度（2026/4〜2027/3）時点：4年=37期 / 3年=38期 / 2年=39期 / 1年=40期**
- 期は入学が新しいほど大きい。新入生は毎年 +1 期。
- 年度は **4月1日で切り替わる**（1〜3月はまだ前年度扱い）。
- 卒業した期（学年が4を超える）は **自動で非表示**。現役＝1〜4年相当の最新4期分だけ表示。

## データモデル（microCMS）

- 部員に数値フィールド **`cohort`（= 期）** を追加。
  - 現メンバーへ一度だけ入力：4年→`37` / 3年→`38` / 2年→`39` / 1年→`40`
  - 以後は不変。新入生のみ登録時に `41`, `42`… を入れる。
- 既存の `year`（学年 1〜4）は削除せず残す（フォールバック用、下記）。

## 計算ロジック（新規 `src/lib/cohort.ts`）

単一責務のユーティリティとして切り出す。日付を引数で受け取れる形にしてテスト可能にする。

```
ANCHOR_FY   = 2026
ANCHOR_4TH  = 37   // 2026年度の4年生の期

// 今日 → 年度（4月1日切替）。monthは1〜12（実装では getMonth()+1）
fiscalYear(date) = month >= 4 ? year : year - 1

// 期 → その年度での学年
grade(cohort, fy) = (fy - ANCHOR_FY) + (ANCHOR_4TH + 4) - cohort
                  // 2026年度: 37期→4, 40期→1

isActive(cohort, fy) = 1 <= grade(cohort, fy) <= 4

label(cohort) = `${cohort}期`
```

検算：fy=2026, cohort=37 → grade=4 / cohort=40 → grade=1 / cohort=36 → grade=5（非表示）。

## 表示の変更

1. **`src/app/roster/page.tsx`**
   - グルーピングキーを学年→**期(`cohort`)** に変更。
   - 表示対象は `isActive` が true の期のみ。
   - 並び順は現状を踏襲し**上級生先頭（＝期の小さい順：37期→40期）**。
   - 見出し帯・アンカータブのラベルを「1ST/2ND…」→「**37期/38期…**」に。アンカーIDは `#cohort-37` 等。

2. **`src/app/roster/[id]/page.tsx`**
   - `{player.year}年生` → **`{cohort}期`** 表示に変更。

3. **`src/app/recruit/page.tsx`（208行目）**
   - `{p.year}年生` → **`{cohort}期`** 表示に変更。

   - `PLAYER_LIST_FIELDS` に `"cohort"` を追加。

## フォールバック（移行を無痛にする）

`cohort` 未入力の部員は、その時点の `year`（学年）から**現在の年度の期を逆算**して表示する：

```
cohortOf(player, fy) = player.cohort ?? ((fy - ANCHOR_FY) + (ANCHOR_4TH + 4) - player.year)
```

- 効果：microCMSをまだ触っていなくても、デプロイ直後の2026年度は正しく「37期…40期」と表示される。
- 注意：このフォールバックは「入力時点の年度」でしか正しくない。**毎年の更新を不要にするには `cohort` の入力が必須**。`cohort` を入れた部員はその瞬間から恒久的に正しくなる。

## テスト方針

- `src/lib/cohort.ts` の純粋関数（`fiscalYear` / `grade` / `isActive` / `cohortOf`）を単体テスト。
  - 年度境界（3/31 と 4/1）、アンカー年（2026各学年）、翌年度（2027で37期が卒業＝非表示）、フォールバック経路。

## 影響範囲外（やらないこと）

- `year` フィールドの削除や、過去メンバー（OB）アーカイブ機能。
- 留年・休部など学年が非標準なケースの特別扱い（必要なら別途 `cohort` 直接入力で対応可能）。
