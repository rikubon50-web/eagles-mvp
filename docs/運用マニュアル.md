# EAGLES HP 運用マニュアル

青山学院大学男子ラクロス部 **EAGLES** 公式サイトの更新方法をまとめたマニュアルです。

更新する場所は **① microCMS** と **② Googleスプレッドシート** の2つだけです。

| 更新したいもの | 使うツール |
|---------------|-----------|
| ニュース（お知らせ・新歓・試合情報） | microCMS → `news` |
| 試合の予定・結果 | microCMS → `games` |
| 部員紹介（ロスター） | microCMS → `players` |
| ブログ | microCMS → `blog` |
| 主将挨拶（チームについて） | microCMS → `about` |
| リーグ順位表（STANDINGS） | Googleスプレッドシート |

> **反映までの時間**：どちらも更新後 **最大5分** でサイトに反映されます。すぐに反映されなくても、5分ほど待ってからページを再読み込みしてください。詳しくは末尾の「反映されないときは」を参照。

---

## 1. microCMS（コンテンツ管理）

文章・写真・試合データなどはすべて microCMS で管理します。

**ログインURL**: https://eagles-mvp.microcms.io
（ログイン情報は管理者から引き継いでください）

ログイン後、左メニューに `news` / `games` / `players` / `blog` / `about` が並んでいます。各メニューを開き、右上の **「追加」** で新規作成、一覧から項目をクリックで編集できます。入力後は必ず右上の **「公開」**（または「公開して保存」）を押してください。下書き保存だけではサイトに表示されません。

---

### 1-1. ニュース（news）

お知らせ・新歓・試合情報を投稿します。トップページとニュース一覧に表示されます。

| 項目 | 内容 |
|------|------|
| タイトル | 記事のタイトル |
| カテゴリ | `お知らせ` / `新歓` / `試合情報` のいずれかを選択 |
| 抜粋（excerpt） | 一覧に出る短い説明（任意。空でも可） |
| 本文 | リッチテキストで自由に入力（写真・リンクも挿入可） |

**新しい記事を追加するには**
1. 左メニューの「news」をクリック
2. 右上の「追加」ボタンを押す
3. タイトル・カテゴリ・本文を入力
4. 右上の「公開」を押す

> カテゴリは必ず3つの中から選んでください。表記が違うと色分け・絞り込みが正しく動きません。

---

### 1-2. 試合スケジュール・結果（games）

トップページの試合一覧、試合ページに表示されます。

| 項目 | 内容 |
|------|------|
| タイトル | 試合名（例：「関東リーグ 第3節」） |
| 開始日時（startAt） | 試合の日時 |
| 会場（venue） | 会場名 |
| ホームチーム名 | 「青山学院大学」など |
| ホームチームロゴ | 画像をアップロード |
| アウェイチーム名 | 相手校名 |
| アウェイチームロゴ | 相手校ロゴ（任意） |
| ステータス（status） | `scheduled`（予定）/ `finished`（終了）/ `postponed`（延期） |
| スコア（自チーム / ourScore） | 試合後に入力 |
| スコア（相手 / oppScore） | 試合後に入力 |
| ボタン文言（ctaText） | 「観戦予約はこちら」などのボタン名（任意） |
| ボタンリンク（ctaUrl） | 上のボタンのリンク先URL（任意） |

> **重要**：試合が「これからの試合」に出るか「結果」に出るかは、**開始日時（startAt）が未来か過去か**で自動的に決まります。日時を正しく入れておけば、当日を過ぎると自動で結果側に移動します。

**新しい試合を登録するには**
1. 「games」→「追加」
2. タイトル・開始日時・会場・チーム名・ロゴを入力
3. ステータスは `scheduled`（予定）のまま「公開」

**試合結果を入力するには**
1. 「games」から該当試合を開く
2. `ステータス` を `finished`（終了）に変更
3. `スコア（自チーム）` と `スコア（相手）` を入力
4. 「公開」（保存）を押す

---

### 1-3. 部員紹介（players）

ロスターページ・トップの部員紹介に表示されます。

| 項目 | 内容 |
|------|------|
| 名前（name） | 氏名（日本語） |
| アルファベット（alphabet） | ローマ字表記 |
| 学年（year） | 数字で入力（1〜4） |
| 写真（photo） | 顔写真をアップロード |
| 学部・学科（faculty） | 任意 |
| 出身高校（highschool） | 任意 |
| 経験スポーツ（sports） | 任意 |
| 一言コメント（comment） | 任意 |
| 好きな言葉・口癖（favoriteWord） | 任意 |
| 憧れの人物（rolemodel） | 任意 |
| 趣味・特技（hobby） | 任意 |
| 生き物に例えると（animal） | 任意 |
| 無人島に1つ持っていくなら（islandItem） | 任意 |
| ラクロスをやってなかったら（alternativePath） | 任意 |
| 先輩の声に表示（isTestimonial） | ONにすると新歓ページの「先輩の声」に掲載されます |

**新入生を追加するには**
1. 「players」→「追加」
2. 名前・学年・写真など各項目を入力して「公開」

**卒業生を削除するには**
1. 対象の部員を開く
2. 右上の「…」→「削除」

> **注意**：学年（year）は自動では上がりません。**毎年4月に、在籍部員の学年を1つずつ手作業で繰り上げて**ください（卒業＝4年生は削除）。

### 1-4. ブログ（blog）

過去記事の閲覧はできますが、**新規投稿はAmeba（アメブロ）側で行う運用**です。サイトへの移行が必要な場合は担当者に相談してください。

---

### 1-5. アバウト（about / 主将挨拶）

「チームについて」ページの **主将挨拶の写真と本文** を管理します。

| 項目 | 内容 |
|------|------|
| ビジュアル（visual） | 主将の写真 |
| 本文（body） | 挨拶文（リッチテキスト） |

主将交代などで内容を変えるときは、`about` を開いて写真・本文を差し替え「公開」してください。

> 「活動場所（地図）」「ラクロスの魅力」「よくある質問（FAQ）」はサイト内に直接書かれており、microCMS/スプレッドシートからは編集できません。変更が必要な場合は担当者に相談してください。

## 2. Googleスプレッドシート（順位表 / STANDINGS）

トップページと STANDINGS ページの順位表は、**1枚のGoogleスプレッドシートと直接つながっています**。
スプレッドシートを書き換えるだけで、サイトに反映されます（最大5分後）。microCMSとは無関係です。

### 仕組み（読むだけでOK）

このシートは「**ファイル → 共有 → ウェブに公開（CSV形式）**」で公開され、その公開リンクをサイトが読み込んでいます。
**いつものシートをそのまま編集すれば自動で反映**されます。新しいシートに作り替えると公開リンクが変わり、サイト側の設定変更（担当者作業）が必要になるので、**シートは作り直さず既存のものを編集**してください。

シートのURL（編集用）は管理者から引き継いでください。

### 列の構成（1行目はヘッダー＝列名。変更不可）

| 列名 | 内容 | 例 |
|------|------|----|
| `block` | ブロック（トップの2列表示の振り分けに使用） | `A` または `B` |
| `rank` | 順位 | `1` |
| `university` | 大学名 | `青山学院大学` |
| `points` | 勝点 | `9` |
| `games` | 試合数 | `3` |
| `gf` | 総得点 | `42` |
| `diff` | 得失点差 | `+18` |
| `updatedAt` | 更新日（サイトに表示される） | `2025/9/15` |

> トップページの順位表は `block` 列の **A / B** で左右2つの表に自動で分かれます。チームごとに必ず `A` か `B` を入れてください。

### 更新手順

1. Googleスプレッドシートを開く
2. 該当チームの行を編集する（順位・勝点・試合数・総得点・得失点差）
3. 新しいチームが増えたら行を追加し、`block` から `diff` まで埋める
4. `updatedAt` 列の日付を今日の日付に更新する
5. **保存ボタンは不要**（スプレッドシートは自動保存）
6. 数分後にサイトを確認する

### やってはいけないこと

- **1行目（列名）を変えない・消さない**：表示が壊れます。
- **「ウェブに公開」を解除しない**：サイトにデータが届かなくなります。
- **新しいシートに作り替えない**：リンクが切れます（編集は既存シートで）。

---

## 3. 反映されないときは

1. **まず5分待って**、ページを再読み込み（スマホは特にキャッシュが残りやすいので、時間をおく／別端末で確認）。
2. **microCMS**：その記事が「公開」状態か確認（下書きだと表示されません）。
3. **スプレッドシート**：編集したのが**サイトとつながっている正しいシート／タブ**か、列名（1行目）を変えていないか確認。
4. それでも反映されない場合は、担当者（サイト管理者）に連絡してください。

> microCMSは、更新時にサイトを即時更新する仕組み（Webhook）も用意されています。設定状況や即時反映の可否は担当者に確認してください。

---

## まとめ

| やること | ツール | 反映 |
|---------|--------|------|
| ニュース投稿 | microCMS → `news` | 最大5分 |
| 試合の予定・結果入力 | microCMS → `games` | 最大5分 |
| 部員の追加・削除 | microCMS → `players` | 最大5分 |
| 主将挨拶の変更 | microCMS → `about` | 最大5分 |
| 順位表の更新 | Googleスプレッドシート | 最大5分 |

困ったときは、操作を確定（microCMSは「公開」、スプレッドシートは自動保存）したうえで5分待つ。それでもダメなら管理者へ。
